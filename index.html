Tarea Nº4

Objetivo:

El propósito de esta tarea es que, en equipos de dos personas, trabajen juntos en un proyecto utilizando Git. Deberán aplicar los comandos básicos de Git (clone, push, pull, merge) y trabajar en diferentes ramas para luego combinarlas en la rama principal (main).

    Instrucciones:
  • Creen un repositorio en GitHub: Uno de los integrantes deberá crear el repositorio y agregar al compañero como colaborador.
  • Clonen el repositorio: Ambos deberán clonar el repositorio en sus computadoras locales.
  • Cree una rama por cada integrante: Cada uno de los integrantes debe crear su propia rama a partir de main (la rama principal). Nombrar la rama con su nombre o un identificador claro (por ejemplo: rama-nombre1 y rama-nombre2).
  • Realicen cambios en sus respectivas ramas: Cada integrante deberá hacer al menos 3 commits en su propia rama, modificando o agregando archivos al proyecto. Pueden agregar contenido como un archivo .md con descripciones, o editar archivos existentes.
  • Hagan push a sus ramas: Una vez que hayan realizado cambios y hecho commits, deberán hacer git push de sus ramas al repositorio remoto en GitHub.
  • Combinación de ramas (Merge): - Una vez que ambos hayan terminado de trabajar en sus ramas, deberán combinar (merge) sus ramas en la rama main. - Primero, uno de los integrantes debe crear un pull request desde su rama hacia main y hacer el merge. - Luego, el otro integrante deberá hacer lo mismo con su rama.
  • Resolución de conflictos (si los hay): Si al intentar combinar las ramas surge un conflicto, deberán resolverlo entre ambos. Esto puede implicar que se coordinen para revisar los archivos en conflicto y hacer ajustes.
  • Entrega: Una vez completada la tarea, deben mandar la URL del repositorio en GitHub donde podamos revisar: - Los commits realizados por ambos. - Las ramas creadas. - La combinación de ramas (merge) en main.

Uso de GitHub por Primera Vez

    Desde la terminal (Comando+Ñ) con GitBash
  1. Escribir: git init
  2. Escribir: git add . (Para que el archivo esté trackeado y se agreguen los cambios)
  3. Escribir: git remote add origin (ingrese link acá del repositorio creado en githu)
  4. Escribir: git branch -m
  5. Escribir: git commit -m "Acá va un comentario/detalle de qué se trata este cambio"
  6. Escribir: git push --set-upstream origin master

Otras indicaciones

  • Para crear una rama: git checkout -b nombrerama
  • Para cambiarme de rama: git checkout nombrerama / git switch nombrerama
  • Si ya se clonó el repositoroio y para actualizar con nuevos archivos/rama: git fetch
  • VSC tiene una interfaz visual para manejar github, se clona el link y en el apartado de source control se pueden agregar los cambios, hacer commits y pushear
